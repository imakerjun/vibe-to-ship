# 스텝 1: 아이디어 선정 + 공유

<Callout type="info">
**이 스텝의 목표** — 각자 **오늘 안에 만들 수 있는 앱 아이디어 1개**를 정하고, 조원들과 간단히 공유합니다.
프롬프트나 코드는 아직 신경 쓰지 마세요. "무엇을 만들지"만 정합니다.
</Callout>

## 어떤 앱을 만들까?

> 이번 주에 **엑셀이나 수작업으로 반복한 일**이 있나요?
> 그걸 앱으로 바꾸면 어떤 모습일까요?

## 진행 방법

모든 작업은 **공유 시트**에서 진행합니다.

1. **후보 적기** — 본인 열의 "아이디어 후보" 칸에 떠오르는 업무 문제/반복 작업을 자유롭게 나열합니다.
2. **하나 고르기** — 그중 오늘 만들고 싶은 아이디어 **하나**를 "핵심 아이디어" 칸에 한 줄로 적습니다.
3. **조원과 공유** — 돌아가며 간단히 읽어주고, 서로 어떤 걸 만들지 알고 넘어갑니다.

<Callout type="default">
조원 투표나 조 대표 아이디어 선정은 따로 하지 않습니다. **각자 본인이 만들고 싶은 앱**을 만들면 됩니다.
</Callout>

## 아이디어 예시

이전 교육에서 동료분들이 실제로 만든 것들입니다. 본인 업무에 맞는 아이디어를 떠올릴 때 참고하세요.

| 유형 | OK금융그룹 실제 사례 | 오늘 만들 수 있는 버전 |
|------|---------------------|----------------------|
| **업무 도구/유틸리티** | 회의실 예약 시스템, 법무사보수 자동계산기, D-day 스케줄 위젯 | 회의실 예약 현황판, 각종 계산기, 일정 D-day 관리 페이지 |
| **고객 응대/상담** | 응대스크립트 웹 뷰어, 고객응대 답변가이드 챗봇 | 자주 쓰는 스크립트/안내문을 검색·복사하는 웹 뷰어 |
| **HR/인사** | HR 평가자 자동매핑, 위탁교육 관리 | 평가 점수 입력 → 등급 자동 계산, 교육 신청 현황 관리 |
| **자산/채권 현황** | IB금융본부 자산현황 대시보드, 이자상환 스케줄 관리 | 매주 확인하는 수치를 한 화면에 모아보는 대시보드 |
| **금융 데이터 조회** | 업무원가율 조회, 저축은행 금리 모니터링 | 자주 보는 데이터를 입력·붙여넣어 조회하는 화면 |
| **보고서/문서 도구** | IB Pre보고 자동 작성, DART 감사보고서 수집·분석 초안 | 정해진 양식에 값을 채우면 보고 초안을 만들어주는 폼 |
| **컴플라이언스/점검** | 공시 알리미, 채권관리 컴플라이언스, 법인카드 점검 | 체크 항목 입력 시 완료 여부를 표시해주는 점검 체크리스트 |

<Callout type="warning">
**범위를 좁히세요!** "전부 다 만들기"보다 "핵심 1가지를 완성"하는 게 훨씬 낫습니다.

- "회의실 예약 전체 시스템" ✕ → "우리 팀 회의실 오늘 예약 현황을 한눈에 보는 페이지" ✓
- "HR 평가 자동화 시스템" ✕ → "평가 점수를 입력하면 등급을 자동 계산해주는 도구" ✓
- "자산 관리 플랫폼" ✕ → "매주 정리하는 핵심 KPI 수치 3개를 카드로 보여주는 대시보드" ✓
</Callout>

## 확인 포인트

> 공유 시트 본인 열의 "핵심 아이디어" 칸이 한 줄로 채워졌고, 조원에게 공유했다면 스텝 1 완료입니다.

## 막혔을 때

| 증상 | 해결 |
|------|------|
| 공유 시트 접속이 안 된다 | 워크숍 채팅의 링크 재확인. 권한 오류면 강사에게 알리기 |
| 아이디어가 안 나온다 | 위 예시 표에서 가장 가까운 유형 하나 고르기 |
| 범위가 너무 크다 | 하루 안에 완성 가능한 수준까지 더 쪼개기 |
