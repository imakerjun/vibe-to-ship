# 스텝 1: 아이디어 선정 + 공유

<Callout type="info">
**이 스텝의 목표** — 조별로 **오늘 안에 만들 수 있는 앱 아이디어 1개**를 정하고, 다른 조와 간단히 공유합니다.
아직 프롬프트나 코드는 신경 쓰지 마세요. 이 스텝에서는 "무엇을 만들지"만 정합니다.
</Callout>

## 어떤 앱을 만들까?

**거창할 필요 없습니다.** 아래 질문에 답해보세요.

> 이번 주에 **엑셀이나 수작업으로 반복한 일**이 있나요?
> 그걸 앱으로 바꾸면 어떤 모습일까요?

## 5분 아이디어 선정법

<Callout type="info">
**이 활동은 공유 스프레드시트에서 진행합니다.**
강사가 안내한 **공유 시트의 조 탭**을 열고, 본인 이름 열(column)에 직접 작성하세요.
다른 조원이 쓰는 내용도 실시간으로 보입니다.
</Callout>

조별로 아래 흐름을 따라 5분 안에 주제를 정합니다. 모든 작업은 **공유 시트 위에서** 진행합니다.

| 시간 | 활동 | 시트에서 하는 일 |
|---|---|---|
| **2분** | 각자 메모 | 본인 열의 "아이디어 후보" 칸에 떠오르는 업무 문제/반복 작업을 줄바꿈으로 자유롭게 나열 |
| **1분** | 돌아가며 공유 | 본인 열의 "핵심 아이디어 1개" 칸에 가장 마음에 드는 하나만 한 줄로 적고, 조원들에게 간단히 읽어주기 |
| **1분** | 조원 투표 | 조원 이름 옆 "투표" 행에 지지하는 아이디어 작성자 이름 적기 (1인 1표, 동률이면 조장이 결정) |
| **1분** | 최종 확정 | 조 대표 행의 "선정된 아이디어"에 옮겨 적고, 스코핑 템플릿 칸(다음 섹션)을 조원들과 함께 채우기 |

<Callout type="default">
**왜 시트에서 하나요?**
- 다른 조의 아이디어가 실시간으로 보여서 서로 영감을 받을 수 있습니다
- 말로 공유할 때보다 **조용한 분도 동등하게 의견을 낼 수 있습니다**
- 나중에 회고/데모 시간에 "그때 누가 뭘 제안했는지" 돌아보기 좋습니다
</Callout>

## 스코핑 템플릿

선정된 아이디어를 **공유 시트 조 탭의 스코핑 영역**에 함께 채워주세요. 이 영역은 한 번만 채우면 되므로 조원이 역할을 나눠 동시에 쓰면 빠릅니다.

```
[앱 이름]:

누가 쓰나: (나? 팀원? 고객?)
뭘 넣나: (입력하는 데이터)
뭘 보나: (화면에 나오는 결과)
저장할 것: (DB에 남길 데이터)
오늘 우선 만들 것: ___만 먼저 만든다
```

## 아이디어 예시

이전 교육에서 동료분들이 실제로 만든 것들입니다. 참고해서 본인 업무에 맞는 아이디어를 떠올려보세요.

| 유형 | OK금융그룹 실제 사례 | 오늘 만들 수 있는 버전 |
|------|---------------------|----------------------|
| **업무 도구/유틸리티** | 회의실 예약 시스템 (14개 실시간 현황), 법무사보수 자동계산기 (취득세·중개수수료), D-day 스케줄 위젯 | 회의실 예약 현황판, 각종 계산기, 일정 D-day 관리 페이지 |
| **고객 응대/상담** | 응대스크립트 웹 뷰어 (엑셀 → 웹 조회+복사), 고객응대 답변가이드 챗봇 | 자주 쓰는 스크립트/안내문을 검색하고 클립보드에 복사하는 웹 뷰어 |
| **HR/인사** | HR 평가자 자동매핑 (이동발령 시 자동 변경), 위탁교육 관리 (PDF→신청자 정보 추출) | 평가 점수 입력 → 등급 자동 계산, 교육 신청 현황 관리 페이지 |
| **자산/채권 현황** | IB금융본부 자산현황 대시보드 (KPI·포트폴리오 차트), 이자상환 스케줄 관리 | 팀이 매주 확인하는 수치를 한 화면에 모아보는 대시보드 |
| **금융 데이터 조회** | 업무원가율 조회 (엑셀 업로드→DB→웹 조회), 저축은행 금리 모니터링 | 자주 보는 데이터를 직접 입력하거나 붙여넣어 조회하는 화면 |
| **보고서/문서 도구** | IB Pre보고 자동 작성, DART 감사보고서 수집·분석 초안 | 정해진 양식에 값을 채워 넣으면 보고 초안을 만들어주는 폼 |
| **컴플라이언스/점검** | 공시 알리미 (법령 기준 자동 분석), 채권관리 컴플라이언스, 법인카드 점검 | 체크 항목 목록을 입력하면 완료 여부를 표시해주는 점검 체크리스트 |

<Callout type="warning">
**범위를 좁히세요!** "전부 다 만들기"보다 "핵심 1가지를 완성"하는 게 훨씬 낫습니다.

**좋은 범위 예시:**
- "회의실 예약 전체 시스템" ✕ → "우리 팀 회의실 오늘 예약 현황을 한눈에 보는 페이지" ✓
- "HR 평가 자동화 시스템" ✕ → "평가 점수를 입력하면 등급을 자동 계산해주는 도구" ✓
- "자산 관리 플랫폼" ✕ → "매주 정리하는 핵심 KPI 수치 3개를 카드로 보여주는 대시보드" ✓
</Callout>

## 조 간 공유 (5분)

5분 아이디어 선정이 끝나면 각 조가 **30초씩** 돌아가며 공유합니다. 다른 조의 시트는 이미 열어볼 수 있으니, 발표 중에 직접 보면서 듣는 걸 추천합니다.

발표는 아래 3가지만 짧게 말하면 됩니다.

- 조 이름
- 앱 이름 한 줄
- "이걸 만들면 우리 팀의 ___가 편해진다"

<Callout type="default">
**왜 공유하나요?** 다른 조의 아이디어를 들으면 "우리 조도 이렇게 좁혀볼까?" 같은 영감이 생깁니다.
또 나중에 데모 시간에 "아, 그 조가 얘기하던 그거구나" 하고 연결되어 몰입도가 높아집니다.
</Callout>

## 확인 포인트

> 조별로 스코핑 템플릿 6칸이 모두 채워졌고, 공유 시트에 반영되었다면 스텝 1 완료입니다.

## 막혔을 때

| 증상 | 해결 |
|------|------|
| 공유 시트에 접속이 안 된다 | 워크숍 채팅에 공유된 링크를 다시 확인. 권한 오류면 강사에게 알리기 |
| 시트에서 내 조 탭이 안 보인다 | 시트 하단 탭 스크롤바를 좌우로 움직여 확인. 그래도 없으면 강사에게 요청 |
| 5분 안에 아이디어가 안 나온다 | 위의 아이디어 예시 표에서 가장 가까운 유형 하나 고르기 |
| 조원 간 의견이 갈린다 | 일단 득표수 많은 쪽으로 진행. 미련 남는 아이디어는 다음 워크숍에서 |
| 범위가 너무 크다 | "오늘 우선 만들 것" 칸을 더 작게 쪼개기. 하루 안에 완성 가능한 수준까지 |
