# 스텝 3: Claude Code로 개선 + 배포

세션 1의 스텝 4와 동일한 흐름입니다. GitHub에 연결하고, VS Code + Claude Code로 개선합니다.

<Steps>

### GitHub 연결

Lovable 상단 **GitHub 아이콘** → **Connect to GitHub** → 저장소 생성

### VS Code에서 코드 가져오기

Claude Code 채팅창에 요청합니다.

```
GitHub에서 내 저장소를 클론해줘.
저장소 주소는 https://github.com/{username}/{repo-name} 이야.
```

### Claude Code로 개선하기

아래 프롬프트 가이드를 참고하여 개선을 요청합니다.

### 2차 배포

```
수정한 코드를 GitHub에 올리고 배포해줘.
```

</Steps>

## 좋은 프롬프트 작성법

Anthropic이 권장하는 프롬프트 작성 원칙입니다. Claude Code에서도 동일하게 적용됩니다.

### 원칙 1: 구체적으로 요청하세요

```
❌ 나쁜 예: UI를 개선해줘.

✅ 좋은 예:
대시보드 상단의 요약 카드를 개선해줘.
- 각 카드에 아이콘을 추가해줘
- 숫자는 크게, 라벨은 작게 표시해줘
- 카드 사이 간격을 16px로 맞춰줘
```

### 원칙 2: 맥락을 알려주세요

```
❌ 나쁜 예: 차트를 추가해줘.

✅ 좋은 예:
이 앱은 자점감사 체크리스트야.
팀장이 월별 완료율 추이를 한눈에 보고 싶어해.
최근 6개월간의 완료율을 꺾은선 그래프로 대시보드에 추가해줘.
```

### 원칙 3: 한 번에 하나씩 요청하세요

```
❌ 나쁜 예: 차트도 추가하고, 필터도 만들고, CSV 내보내기도 해줘.

✅ 좋은 예:
먼저 월별 완료율 차트를 추가해줘.
(완료 후) 카테고리별 필터 기능을 추가해줘.
(완료 후) 데이터를 CSV로 내보내는 버튼을 추가해줘.
```

<Callout type="info">
**복잡한 요청은 쪼개세요.** Claude Code는 한 번에 하나씩 요청할 때 가장 정확하게 작업합니다.
큰 변경을 한꺼번에 요청하면 의도와 다른 결과가 나올 수 있습니다.
</Callout>

## 개선 요청 예시

본인의 앱에 맞게 골라서 시도해보세요.

### 기능 추가

```
이 앱의 데이터를 CSV 파일로 내보내는 기능을 추가해줘.
다운로드 버튼을 목록 상단에 배치해줘.
```

### UI 개선

```
체크리스트 카드의 디자인을 개선해줘.
- 완료 항목은 배경색을 연하게
- 미완료 항목은 좌측에 강조 바(accent bar) 추가
- 카드 간 간격을 12px로
```

### 데이터 시각화

```
이 앱에서 관리하는 점검 데이터를 기반으로
카테고리별 완료율을 보여주는 막대 그래프를 추가해줘.
대시보드 상단에 배치해줘.
```

## 확인 포인트

> Claude Code로 개선한 기능이 배포된 URL에서 동작하면 스텝 3 완료입니다.

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| GitHub 연결 버튼이 작동하지 않음 | Lovable이 GitHub 권한을 받지 못한 상태 | Lovable 설정에서 GitHub 연동을 해제하고 다시 연결. 브라우저 팝업 차단 해제 확인 |
| Clone 후 VS Code에 파일이 안 보임 | 폴더를 열지 않았거나 잘못된 경로에 클론된 경우 | VS Code에서 File → Open Folder로 클론된 폴더를 직접 열기. 클론 경로는 Claude Code가 알려준 경로 확인 |
| Claude Code가 프로젝트 구조를 못 파악함 | 프로젝트 루트가 아닌 상위 폴더가 열려 있는 경우 | `이 프로젝트의 구조를 설명해줘`로 먼저 확인. 이상하면 `package.json이 있는 폴더를 열어줘`라고 요청 |
| 코드 수정 후 빌드 에러가 발생함 | 여러 변경을 한꺼번에 요청해 충돌이 발생한 경우 | `지금 에러 메시지를 보여줄게, 원인을 찾아서 고쳐줘`라고 에러 내용을 그대로 붙여넣기. 한 번에 하나씩만 요청 |
| 배포 후 브라우저에서 빈 화면이 보임 | 빌드는 성공했지만 런타임 오류가 발생한 경우 | 브라우저 개발자 도구(F12) → Console 탭의 빨간 에러 메시지를 Claude Code에 붙여넣고 `이 오류를 고쳐줘`라고 요청 |
| Lovable과 Claude Code 변경사항이 충돌함 | 양쪽에서 동시에 같은 파일을 수정한 경우 | 작업 순서를 정해서 진행. Claude Code로 수정 중에는 Lovable에서 편집하지 않기. 충돌 시 아래 복구 방법 참고 |

<Callout type="warning">
**코드가 깨졌을 때 복구 방법**

**방법 1: git으로 되돌리기 (Claude Code에서 요청)**

```
방금 수정하기 전 상태로 코드를 되돌려줘.
```

직접 명령이 필요하면 Claude Code 채팅창에 요청합니다.

```
git checkout으로 마지막 커밋 상태로 되돌려줘.
```

**방법 2: Lovable 히스토리로 되돌리기**

1. Lovable 좌측 사이드바에서 **History** 패널을 엽니다
2. 정상이었던 시점의 항목을 찾아 클릭합니다
3. **Restore this version**을 선택하면 그 시점 코드로 복원됩니다
4. 복원 후 GitHub에 자동 반영되므로 Claude Code에서도 최신 코드를 다시 가져옵니다

어느 방법이든 막히면 퍼실리테이터를 불러주세요.
</Callout>
