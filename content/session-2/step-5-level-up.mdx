# 스텝 5: 자산화 — 재사용할 수 있게 만들기

스텝 4까지 만든 앱은 **동작하는 결과물**입니다.
이제 한 단계 더 — 오늘 했던 작업을 **다음에도 쓸 수 있게** 정리합니다.

## Claude Code 핵심 기능 맵

Claude Code에는 작업을 효율적으로 만들어주는 기능들이 있습니다.

## 1단계: CLAUDE.md 만들기

**CLAUDE.md**는 Claude Code에게 "이 프로젝트는 이런 프로젝트야"라고 알려주는 파일입니다.
한 번 작성하면 매번 반복 설명하지 않아도 됩니다.

```
이 프로젝트의 CLAUDE.md 파일을 만들어줘.

아래 내용을 포함해줘:
- 프로젝트 목적: {한 줄 설명}
- 사용 기술: React, TypeScript, Tailwind, Supabase
- 주요 파일 구조
- 코딩 규칙 (있다면)
```

<Callout type="info">
**왜 중요한가?** CLAUDE.md가 있으면 Claude Code가 프로젝트를 이해한 상태에서 시작합니다.
"이 프로젝트는 React야", "DB는 Supabase야" 같은 설명을 매번 하지 않아도 됩니다.
</Callout>

## 2단계: 기본 슬래시 커맨드 체험하기

Claude Code 채팅창에서 `/`를 입력하면 사용 가능한 명령어 목록이 나옵니다. 우선 내장 커맨드 몇 개를 가볍게 써보세요.

| 커맨드 | 하는 일 | 언제 쓰나 |
|--------|---------|----------|
| `/init` | 프로젝트 분석 후 CLAUDE.md 자동 생성 | 프로젝트를 처음 열 때 |
| `/compact` | 지금까지의 대화 내용을 요약·정리 | 대화가 길어졌을 때 |
| `/review` | 변경된 코드 리뷰 | 배포 전 검토 |
| `/help` | 사용 가능한 기능 목록 | 뭘 할 수 있는지 모를 때 |

## 3단계: 커스텀 슬래시 커맨드 1개 만들기

반복되는 프롬프트를 **`.claude/commands/{이름}.md`** 파일로 저장하면 `/이름` 으로 호출할 수 있는 나만의 커맨드가 됩니다. 파일 상단의 YAML frontmatter로 설명·인자 힌트·허용 툴을 지정할 수 있고, 본문에는 Claude에게 전달될 프롬프트를 적습니다. 인자는 `$ARGUMENTS` 로 받습니다.

오늘 가장 자주 한 작업 하나를 떠올려 보세요. 예: "변경 사항을 정리해 커밋 메시지까지 만들어 배포하기". 그걸 커맨드로 만들어 봅시다.

Claude Code에 이렇게 요청하세요:

```
아래 스펙으로 커스텀 슬래시 커맨드 1개를 만들어줘.

파일 경로: .claude/commands/ship.md
이름: /ship
하는 일: 현재 변경된 파일을 요약하고, 의미 있는 커밋 메시지를 만들어
         커밋·푸시한 뒤, Lovable/Vercel에 반영되었는지 확인한다.
frontmatter: description, argument-hint 포함
본문: Claude에게 전달할 지시 프롬프트 (한국어)
```

생성되면 Claude Code에서 `/ship` 을 입력해 바로 실행해보세요.

<Callout type="default">
**공식 문서** — [Slash commands (docs.claude.com)](https://docs.claude.com/en/docs/claude-code/slash-commands)
프로젝트 커맨드는 `.claude/commands/` 에, 개인 커맨드는 `~/.claude/commands/` 에 저장됩니다. 팀에 공유하려면 프로젝트 폴더를 Git에 커밋하면 됩니다.
</Callout>

## 4단계: 스킬(Skill) 1개 만들기

**스킬**은 커맨드보다 한 단계 위의 재사용 단위입니다. `SKILL.md` 한 파일에 **언제 발동되는지(`description`)와 어떻게 수행하는지(본문 지시문)** 를 담아두면, Claude가 상황에 맞게 알아서 불러옵니다. 다단계 워크플로우나 규칙이 많은 작업에 적합합니다.

구조는 다음과 같습니다.

```
.claude/skills/
└── workshop-report/
    └── SKILL.md
```

`SKILL.md` 예시:

```markdown
---
name: workshop-report
description: 오늘 만든 앱의 기능·화면·DB 스키마를 요약한
  워크숍 회고 리포트를 생성할 때 사용합니다.
  사용자가 "회고 리포트", "워크숍 정리", "오늘 만든 것 요약"을
  요청하면 발동됩니다.
---

# Workshop Report Skill

다음 순서로 워크숍 회고 리포트를 작성한다.

1. 현재 프로젝트의 주요 화면과 기능을 나열한다.
2. Supabase 스키마(테이블·컬럼)를 요약한다.
3. 오늘 겪은 막힌 지점과 해결 방법을 3줄 이내로 정리한다.
4. 결과물을 `docs/workshop-report.md` 로 저장한다.
```

Claude Code에 이렇게 요청하면 위 파일을 만들 수 있습니다.

```
아래 스펙으로 스킬을 만들어줘.

경로: .claude/skills/workshop-report/SKILL.md
frontmatter: name, description (발동 조건을 구체적으로)
본문: 워크숍 결과를 요약해 docs/workshop-report.md 에 저장하는
      단계별 지시문 (한국어, 4단계 이내)
```

만든 뒤 "워크숍 회고 리포트 만들어줘" 라고 말해보면 스킬이 자동으로 선택됩니다.

<Callout type="default">
**공식 문서** — [Skills (docs.claude.com)](https://docs.claude.com/en/docs/claude-code/skills)
`description` 은 "언제 이 스킬을 써야 하는지"를 Claude가 읽고 판단하는 근거이므로, 트리거 상황과 키워드를 구체적으로 적을수록 정확히 발동됩니다.
</Callout>

## 5단계: 최종 배포

정리가 끝나면 최종 배포합니다.

```
CLAUDE.md, .claude/commands/ship.md, .claude/skills/workshop-report/SKILL.md를
포함해서 변경 사항을 GitHub에 올리고 배포해줘.
```

## 확인 포인트

> CLAUDE.md, 커스텀 슬래시 커맨드 1개, 스킬 1개가 만들어지고 커밋되면 스텝 5 완료입니다.
