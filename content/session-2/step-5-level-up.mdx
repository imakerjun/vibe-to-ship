# 스텝 5: 자산화 — 재사용할 수 있게 만들기

스텝 4까지 만든 앱은 **동작하는 결과물**입니다.
이제 한 단계 더 — 오늘 했던 작업을 **다음에도 쓸 수 있게** 정리합니다.

## Claude Code 핵심 기능 맵

Claude Code에는 작업을 효율적으로 만들어주는 기능들이 있습니다.

| 기능 | 한 줄 설명 | 공식 문서 |
|---|---|---|
| **CLAUDE.md** | 프로젝트 맥락을 한 번 적어두면 매번 설명할 필요가 없음 | [Memory](https://docs.claude.com/en/docs/claude-code/memory) |
| **Slash Commands** | `/init`, `/compact` 같은 내장 커맨드 + 내가 만든 `/배포` 같은 커스텀 커맨드 | [Slash commands](https://docs.claude.com/en/docs/claude-code/slash-commands) |
| **Skills** | 다단계 워크플로우를 묶어 둔 재사용 단위 | [Skills](https://docs.claude.com/en/docs/claude-code/skills) |
| **Hooks** | 특정 이벤트에 자동으로 실행되는 동작 (예: 파일 저장 시) | [Hooks](https://docs.claude.com/en/docs/claude-code/hooks) |

## 1단계: CLAUDE.md 만들기

**CLAUDE.md**는 Claude Code에게 "이 프로젝트는 이런 프로젝트야"라고 알려주는 파일입니다. 한 번 만들어두면 매번 반복 설명하지 않아도 됩니다.

### 먼저 `/init` 으로 자동 생성

직접 프롬프트를 쓸 필요 없이, Claude Code 채팅창에 아래 한 줄만 입력하면 프로젝트를 분석해 CLAUDE.md 초안을 자동으로 만들어줍니다.

```
/init
```

실행이 끝나면 프로젝트 루트에 `CLAUDE.md` 가 생성되어 있습니다. 한 번 열어서 내용이 내 프로젝트와 맞는지만 가볍게 훑어보세요.

### 작업이 끝나면 Claude Code에게 회고를 시키기

CLAUDE.md를 잘 관리하는 가장 쉬운 방법은 **작업이 한 단락 끝났을 때 Claude Code에게 직접 물어보는 것**입니다. 파일을 열어 편집하거나 마크다운 문법을 알 필요가 없습니다.

작업을 한 차례 마쳤다면 Claude Code 채팅창에 이렇게 요청해보세요.

```
오늘 우리가 한 작업을 돌아봤을 때,
내가 매번 반복해서 너에게 설명하거나 부탁한 게 있었을까?

만약 있다면 CLAUDE.md에 새로 추가하거나 업데이트하면 좋을 내용을
정리해서 보여줘. 내가 확인하고 OK하면 그때 파일에 반영해줘.
```

Claude Code가 대화를 되짚으며 "이런 건 매번 부탁하셨던데, 규칙으로 넣을까요?" 같은 제안을 줍니다. 마음에 드는 항목만 골라 OK 하면 알아서 CLAUDE.md를 업데이트합니다.

이렇게 **회고 → 규칙화** 흐름을 반복하면, CLAUDE.md가 본인의 작업 패턴에 맞춰 자라납니다. 비개발자도 마크다운을 한 줄도 쓰지 않고 자기만의 작업 환경을 다듬을 수 있습니다.

<Callout type="default">
**공식 문서** — [Manage Claude's memory (docs.claude.com)](https://docs.claude.com/en/docs/claude-code/memory)
CLAUDE.md는 Claude Code가 대화 시작마다 자동으로 읽는 프로젝트 메모리 파일입니다. 한 번 적은 내용은 이후 모든 대화에 영향을 줍니다.
</Callout>

## 2단계: 기본 슬래시 커맨드 체험하기

Claude Code 채팅창에서 `/`를 입력하면 사용 가능한 명령어 목록이 나옵니다. 우선 내장 커맨드 몇 개를 가볍게 써보세요.

| 커맨드 | 하는 일 | 언제 쓰나 |
|--------|---------|----------|
| `/init` | 프로젝트 분석 후 CLAUDE.md 자동 생성 | 프로젝트를 처음 열 때 |
| `/compact` | 지금까지의 대화 내용을 요약·정리 | 대화가 길어졌을 때 |
| `/review` | 변경된 코드 리뷰 | 배포 전 검토 |
| `/help` | 사용 가능한 기능 목록 | 뭘 할 수 있는지 모를 때 |

## 3단계: 커스텀 슬래시 커맨드 1개 만들기

반복되는 프롬프트를 **`.claude/commands/{이름}.md`** 파일로 저장하면 `/이름` 으로 호출할 수 있는 나만의 커맨드가 됩니다. 파일 상단의 YAML frontmatter로 설명·인자 힌트·허용 툴을 지정할 수 있고, 본문에는 Claude에게 전달될 프롬프트를 적습니다. 인자는 `$ARGUMENTS` 로 받습니다.

CLAUDE.md를 업데이트할 때처럼, 커스텀 커맨드도 **작업이 한 단락 끝났을 때 Claude Code에게 회고를 시키는 방식**이 가장 쉽습니다. 어떤 프롬프트를 커맨드로 만들지 직접 고르지 않아도 됩니다.

```
오늘 우리가 한 작업을 돌아봤을 때,
내가 비슷한 흐름으로 여러 번 부탁한 일이 있었을까?

만약 있다면 그걸 슬래시 커맨드로 만들어두면 좋을 것 같은데,
어떤 커맨드를 만들면 좋을지 후보를 정리해서 보여줘.
이름·하는 일·언제 쓰면 좋은지 함께 알려주고,
내가 OK 한 것만 실제 파일로 만들어줘.
```

Claude Code가 대화를 되짚어 "예: `/배포` — 변경사항을 커밋하고 푸시한 뒤 Lovable 반영까지 확인" 같은 후보를 제안합니다. 마음에 드는 것만 골라 OK 하면 알아서 `.claude/commands/배포.md` 파일을 만들어줍니다.

커맨드 이름은 **한국어**로도 만들 수 있습니다. 영어권 개발 문화에서는 배포를 보통 `/ship`("출항하다 → 출시하다")처럼 표현하지만, 본인이 평소에 쓰는 말로 `/배포` 라고 지으면 나중에 무엇을 하는 커맨드였는지 더 빨리 떠올릴 수 있어 기억하고 자주 쓰기에 편합니다. Claude Code에게 회고를 시킬 때 "이름은 한국어로 지어줘"라고 한 줄만 덧붙이면 됩니다.

이후에는 Claude Code 채팅창에 `/배포` 한 줄만 입력하면 똑같은 일이 반복됩니다.

<Callout type="default">
**공식 문서** — [Slash commands (docs.claude.com)](https://docs.claude.com/en/docs/claude-code/slash-commands)
프로젝트 커맨드는 `.claude/commands/` 에, 개인 커맨드는 `~/.claude/commands/` 에 저장됩니다. 팀에 공유하려면 프로젝트 폴더를 Git에 커밋하면 됩니다.
</Callout>

## 4단계: 스킬(Skill) 1개 만들기

**스킬**은 커맨드보다 한 단계 위의 재사용 단위입니다. 단순한 한 번의 요청이 아니라 **여러 단계로 이루어진 작업**을 묶어두면, 비슷한 상황이 오면 Claude가 알아서 그 절차를 떠올려 따라갑니다. 다단계 워크플로우나 규칙이 많은 작업에 적합합니다.

스킬도 회고로 만드는 게 가장 쉽습니다. 어떤 절차를 스킬로 만들지 직접 고르지 않아도 됩니다.

```
오늘 우리가 한 작업을 돌아봤을 때,
여러 단계를 거쳐서 진행했던 작업이나
"이런 상황이 오면 항상 이렇게 처리해줘" 라고 할 만한 게 있었을까?

만약 있다면 그걸 스킬로 만들어두면 좋을 것 같은데,
어떤 스킬을 만들면 좋을지 후보를 정리해서 보여줘.
이름·언제 발동되는지·어떤 절차를 따르는지 함께 알려주고,
내가 OK 한 것만 실제 파일로 만들어줘.
```

Claude Code가 "예: 워크숍 회고 리포트 — 화면/기능/DB 스키마/막힌 점을 정리해 `docs/workshop-report.md` 로 저장" 같은 후보를 제안합니다. OK 한 것만 `.claude/skills/{이름}/SKILL.md` 파일로 만들어줍니다.

만든 뒤에는 평소처럼 "워크숍 회고 리포트 만들어줘" 라고 말하기만 하면, Claude가 이 스킬이 맞겠다 싶을 때 자동으로 그 절차를 따라갑니다.

<Callout type="default">
**공식 문서** — [Skills (docs.claude.com)](https://docs.claude.com/en/docs/claude-code/skills)
스킬은 "언제 발동될지"를 Claude가 스스로 판단해서 호출하는 점이 슬래시 커맨드와 가장 큰 차이입니다.
</Callout>

## 5단계: 최종 배포

정리가 끝나면 최종 배포합니다.

```
오늘 만든 CLAUDE.md, /배포 커맨드 파일, 워크숍 보고서 스킬 파일을
모두 포함해서 GitHub에 올리고 배포해줘.
```

## 확인 포인트

> CLAUDE.md, 커스텀 슬래시 커맨드 1개, 스킬 1개가 만들어지고 커밋되면 스텝 5 완료입니다.
