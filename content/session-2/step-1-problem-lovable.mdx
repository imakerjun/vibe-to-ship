# 스텝 1: 업무 파악 + Lovable MVP

## 어떤 앱을 만들까?

**거창할 필요 없습니다.** 아래 질문에 답해보세요.

> 이번 주에 **엑셀이나 수작업으로 반복한 일**이 있나요?
> 그걸 앱으로 바꾸면 어떤 모습일까요?

### 스코핑 템플릿

조별로 아래를 채워주세요. **오늘 안에 가능한 범위**에 집중합니다.

```
[앱 이름]:

누가 쓰나: (나? 팀원? 고객?)
뭘 넣나: (입력하는 데이터)
뭘 보나: (화면에 나오는 결과)
저장할 것: (DB에 남길 데이터)
오늘 우선 만들 것: ___만 먼저 만든다
```

### 아이디어 예시 (사전 설문 기반)

| 유형 | 예시 |
|------|------|
| **점검/감사** | 자점감사 체크리스트, 법인카드 점검, 규정 변경 확인 |
| **데이터 취합** | 인력 현황 집계, 부서별 업무 현황, 성과 평가 취합 |
| **보고서 자동화** | 해외법인 보고서 번역, 감사 결과 보고서, 실적 대시보드 |
| **업무 도구** | 여신금리 계산기, 회의실 예약, FAQ 검색 |

<Callout type="warning">
**범위를 좁히세요!** "전부 다 만들기"보다 "핵심 1가지를 완성"하는 게 훨씬 낫습니다.
예: "인력 현황 전체 시스템" ✕ → "우리 팀 인원 현황을 한눈에 보는 대시보드" ✓
</Callout>

## Lovable MVP 만들기

세션 1의 스텝 1과 동일한 흐름입니다. 이번에는 **프롬프트를 직접 작성**합니다.

### 프롬프트 작성 가이드

좋은 프롬프트는 **구체적이고 맥락이 풍부**합니다. 아래 구조를 참고하세요.

```
{앱 이름} 웹앱을 만들어줘.

누가 쓰나: {사용자 설명}
목적: {이 앱이 해결하는 문제}

핵심 기능:
- {기능 1}: {구체적인 동작 설명}
- {기능 2}: {구체적인 동작 설명}

샘플 데이터:
- {실제 업무에서 나올 법한 예시 데이터}

디자인:
- 깔끔한 카드 UI, 다크 모드
- 모바일 반응형
```

<Callout type="info">
**프롬프트 팁**
- 기능은 **3개 이하**로 시작하세요. 핵심 1개를 먼저 만들고, 채팅으로 하나씩 추가하세요
- "예쁘게 해줘" 보다 **"카드 형태로 보여줘"** 처럼 구체적으로 요청하세요
- 샘플 데이터를 넣으면 결과물을 바로 확인할 수 있어 훨씬 효과적입니다
</Callout>

## 확인 포인트
