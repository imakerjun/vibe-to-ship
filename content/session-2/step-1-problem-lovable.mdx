# 스텝 1: 업무 파악 + Lovable MVP

## 어떤 앱을 만들까?

**거창할 필요 없습니다.** 아래 질문에 답해보세요.

> 이번 주에 **엑셀이나 수작업으로 반복한 일**이 있나요?
> 그걸 앱으로 바꾸면 어떤 모습일까요?

### 스코핑 템플릿

조별로 아래를 채워주세요. **오늘 안에 가능한 범위**에 집중합니다.

```
[앱 이름]:

누가 쓰나: (나? 팀원? 고객?)
뭘 넣나: (입력하는 데이터)
뭘 보나: (화면에 나오는 결과)
저장할 것: (DB에 남길 데이터)
오늘 우선 만들 것: ___만 먼저 만든다
```

### 아이디어 예시

이전 교육에서 동료분들이 실제로 만든 것들입니다. 참고해서 본인 업무에 맞는 아이디어를 떠올려보세요.

| 유형 | 실제 만들어진 예시 | 핵심 패턴 |
|------|-------------------|----------|
| **서류 자동 처리** | 여신서류 대조 체크리스트, 모기지 심사 자동 계산, 스캔 서류 자동 분류 | 엑셀/PDF 업로드 → 자동 검증·분류 |
| **데이터 수집·분석** | 저축은행 금리 모니터링, 분기 실적 자동 수집, 업무원가율 조회 | 여러 사이트 데이터 → 대시보드 |
| **보고서 자동화** | AI 미디어센터 (보도자료 초안), IB Pre보고 자동 작성, DART 감사보고서 수집 | 데이터 → 보고서 초안 자동 생성 |
| **점검/감사** | 인감 검증 자동화, 공시 알리미, 채권관리 컴플라이언스, 법인카드 점검 | 규정 기준 자동 검증 |
| **일정/관리 도구** | 회의실 예약 시스템, 이자상환 스케줄 관리, D-day 위젯, 동호회 지원금 자동화 | 반복 관리 업무 → 웹 대시보드 |
| **HR/인사** | 평가자 자동매핑, 면접관 평가 관리, 위탁교육 관리, 급여 품의 자동화 | 수기 취합 → 자동 처리 |
| **고객 응대** | 상담 스크립트 웹 뷰어, 고객응대 답변가이드 챗봇 | FAQ → 빠른 검색·복사 |

<Callout type="warning">
**범위를 좁히세요!** "전부 다 만들기"보다 "핵심 1가지를 완성"하는 게 훨씬 낫습니다.

**좋은 범위 예시:**
- "모기지 심사 전체 시스템" ✕ → "심사 체크리스트 항목을 한눈에 보는 대시보드" ✓
- "금리 모니터링 플랫폼" ✕ → "우리 팀이 보는 5개 은행 금리를 한 페이지에 정리" ✓
- "인사 평가 시스템" ✕ → "평가 점수를 입력하면 등급을 자동 계산해주는 도구" ✓
</Callout>

## Lovable MVP 만들기

세션 1의 스텝 1과 동일한 흐름입니다. 이번에는 **프롬프트를 직접 작성**합니다.

### 프롬프트 작성 가이드

좋은 프롬프트는 **구체적이고 맥락이 풍부**합니다. 아래 구조를 참고하세요.

```
{앱 이름} 웹앱을 만들어줘.

누가 쓰나: {사용자 설명}
목적: {이 앱이 해결하는 문제}

핵심 기능:
- {기능 1}: {구체적인 동작 설명}
- {기능 2}: {구체적인 동작 설명}

샘플 데이터:
- {실제 업무에서 나올 법한 예시 데이터}

디자인:
- 깔끔한 카드 UI, 다크 모드
- 모바일 반응형
```

<Callout type="info">
**프롬프트 팁**
- 기능은 **3개 이하**로 시작하세요. 핵심 1개를 먼저 만들고, 채팅으로 하나씩 추가하세요
- "예쁘게 해줘" 보다 **"카드 형태로 보여줘"** 처럼 구체적으로 요청하세요
- 샘플 데이터를 넣으면 결과물을 바로 확인할 수 있어 훨씬 효과적입니다
</Callout>
