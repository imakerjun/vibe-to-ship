# 스텝 4: 자산화 — 재사용할 수 있게 만들기

스텝 3까지 만든 앱은 **동작하는 결과물**입니다.
이제 한 단계 더 — 오늘 했던 작업을 **다음에도 쓸 수 있게** 정리합니다.

## Claude Code 핵심 기능 맵

Claude Code에는 작업을 효율적으로 만들어주는 기능들이 있습니다.

## 1단계: CLAUDE.md 만들기

**CLAUDE.md**는 Claude Code에게 "이 프로젝트는 이런 프로젝트야"라고 알려주는 파일입니다.
한 번 작성하면 매번 반복 설명하지 않아도 됩니다.

```
이 프로젝트의 CLAUDE.md 파일을 만들어줘.

아래 내용을 포함해줘:
- 프로젝트 목적: {한 줄 설명}
- 사용 기술: React, TypeScript, Tailwind, Supabase
- 주요 파일 구조
- 코딩 규칙 (있다면)
```

<Callout type="info">
**왜 중요한가?** CLAUDE.md가 있으면 Claude Code가 프로젝트를 이해한 상태에서 시작합니다.
"이 프로젝트는 React야", "DB는 Supabase야" 같은 설명을 매번 하지 않아도 됩니다.
</Callout>

## 2단계: 슬래시 커맨드 체험하기

Claude Code 채팅창에서 `/`를 입력하면 사용 가능한 명령어 목록이 나옵니다.

| 커맨드 | 하는 일 | 언제 쓰나 |
|--------|---------|----------|
| `/init` | 프로젝트 분석 후 CLAUDE.md 자동 생성 | 프로젝트를 처음 열 때 |
| `/compact` | 지금까지의 대화 내용을 요약·정리 | 대화가 길어졌을 때 |
| `/review` | 변경된 코드 리뷰 | 배포 전 검토 |
| `/help` | 사용 가능한 기능 목록 | 뭘 할 수 있는지 모를 때 |

직접 해보세요:
```
/init
```

## 3단계: 오늘 작업의 에센스 추출

오늘 만든 앱에서 **재사용할 수 있는 패턴**을 정리합니다.

Claude Code에게 요청하세요:

```
오늘 이 프로젝트에서 작업한 내용을 분석해서,
다음에 비슷한 앱을 만들 때 재사용할 수 있는
커스텀 커맨드를 2~3개 만들어줘.

예를 들어:
- 체크리스트 앱 초기 세팅
- Supabase DB 연동 설정
- 대시보드 차트 추가

.claude/commands/ 디렉터리에 마크다운 파일로 저장해줘.
```

<Callout type="default">
**커스텀 커맨드**는 자주 하는 작업을 한 번의 명령어로 실행할 수 있게 만든 것입니다.
`.claude/commands/` 폴더에 마크다운 파일로 저장하면, Claude Code에서 `/` + 파일명으로 호출할 수 있습니다.
팀원에게 이 폴더를 공유하면 같은 워크플로우를 함께 쓸 수 있습니다.
</Callout>

## 4단계: 최종 배포

정리가 끝나면 최종 배포합니다.

```
CLAUDE.md와 커스텀 커맨드를 포함해서
변경 사항을 GitHub에 올리고 배포해줘.
```

## 확인 포인트

> CLAUDE.md가 생성되고, 커스텀 커맨드 1개 이상이 만들어지면 스텝 4 완료입니다.
