# 스텝 3: DB + 기능 확장 + 배포

<span className="step-tools">Lovable Cloud</span>

**이 스텝에서 하는 일**: 스텝 2에서 만든 Lovable MVP에 DB와 구글 로그인을 연결하고, 추가 기능을 보완한 뒤 1차 배포 URL을 생성합니다.

<Callout type="info">
**세션 1을 참고하세요.**
DB 연동과 구글 로그인의 상세 가이드는 [스텝 2: DB 연동](/session-1/step-2-db)과 [스텝 3: 구글 로그인](/session-1/step-3-google-login)에 있습니다.
이 페이지는 핵심 단계를 빠르게 재기술하므로, 막히면 세션 1 문서를 다시 확인하세요.
</Callout>

---

## 진행 순서

<Steps>

### DB 연동하기

아래 프롬프트를 입력하세요. 내 앱의 테이블명과 컬럼명에 맞게 수정합니다.

```
Lovable Cloud를 사용해서 데이터를 저장해줘.

저장할 데이터:
- {테이블명}: {컬럼 목록과 용도}

동작:
- 현재 하드코딩된 데이터를 DB에서 가져오도록 변경
- {사용자 액션}하면 DB에 저장
- 새로고침해도 데이터가 유지되어야 함
```

**동작 확인:**
1. 데이터를 입력하거나 상태를 변경합니다
2. 브라우저를 **새로고침**합니다
3. 데이터가 그대로 유지되는지 확인합니다

<Callout type="warning">
**Lovable Cloud가 활성화되지 않았다면**
좌측 패널에서 **Cloud** 탭을 클릭하고 활성화 버튼을 누른 뒤 다시 시도하세요.
</Callout>

### 구글 로그인 추가하기

사용자 구분이 필요하면 아래 프롬프트를 입력하세요.

```
구글 로그인 기능을 추가해줘.

1. 로그인하지 않은 사용자에게는 로그인 버튼만 보여줘
2. 로그인한 사용자만 앱을 사용할 수 있게 해줘
3. 사용자별로 데이터가 분리되어야 해
4. 상단에 로그인한 사용자 이름과 프로필 사진 표시
5. 로그아웃 버튼도 추가해줘
```

<Callout type="info">
**Lovable Cloud의 Managed Google Auth**
Google Cloud 콘솔에서 별도로 OAuth 클라이언트를 만들 필요 없습니다.
프롬프트 한 줄이면 구글 로그인이 활성화됩니다.
</Callout>

### 추가 기능 요청하기 (1~2건)

핵심 기능 중 빠진 것을 보완합니다. 예시:

```
다음 기능을 추가해줘.
- {기능 1 설명}: {동작 방식}
- {기능 2 설명}: {동작 방식}
```

**추가 기능 아이디어:**
- 데이터 내보내기 (CSV 다운로드)
- 검색 및 필터 기능
- 상태 변경 히스토리 표시
- 알림 또는 마감일 표시

### 1차 배포하기

Lovable 우측 상단 **Publish** 버튼을 클릭합니다.

1. **Publish** 버튼 클릭
2. 배포 완료 후 생성된 URL을 복사합니다
3. 새 탭에서 URL을 열어 앱이 정상 동작하는지 확인합니다

<Callout type="default" emoji="🔗">
배포 URL은 팀원과 공유하거나 데모에 바로 사용할 수 있습니다.
URL을 메모해 두세요 — 스텝 3에서 계속 사용합니다.
</Callout>

</Steps>

## 검증 단계 (5단계)

배포 후 아래 순서대로 확인하세요.

| 단계 | 테스트 항목 | 기대 결과 |
|------|------------|----------|
| 1 | 배포 URL 접속 | 앱 화면이 정상적으로 로드된다 |
| 2 | 구글 로그인 | 로그인 후 앱 메인 화면으로 이동한다 |
| 3 | 데이터 입력 및 저장 | 입력한 데이터가 화면에 반영된다 |
| 4 | 새로고침 | 데이터가 사라지지 않고 그대로 유지된다 |
| 5 | 로그아웃 → 재로그인 | 이전에 저장한 데이터가 다시 보인다 |

> 5단계를 모두 통과하면 스텝 2 완료입니다.

## 트러블슈팅

| 증상 | 원인 | 해결 |
|------|------|------|
| DB 연동 후 데이터가 안 보임 | 데이터 불러오기 로직 누락 | "DB에서 데이터를 불러오는 부분을 확인해줘"라고 요청 |
| 새로고침하면 데이터 사라짐 | 저장 로직이 메모리에만 동작 | "저장 로직이 제대로 동작하는지 확인해줘"라고 요청 |
| Lovable Cloud 활성화 안내 메시지 | Cloud 기능 미활성화 | 좌측 패널 Cloud 탭에서 활성화 버튼 클릭 |
| 구글 로그인 버튼이 안 보임 | 인증 컴포넌트 미렌더링 | "구글 로그인 버튼이 안 보이는데 확인해줘"라고 요청 |
| 로그인 후 빈 화면 | 로그인 상태에 따른 조건부 렌더링 오류 | "로그인 후 데이터를 불러오는 부분을 확인해줘"라고 요청 |
| 배포 URL에서만 로그인이 안 됨 | 허용된 리디렉션 URL 미설정 | "배포 URL에서 구글 로그인이 안 되는 이유를 찾아줘"라고 요청 |
| 로그인 팝업이 차단됨 | 브라우저 팝업 차단 설정 | 브라우저 팝업 차단 해제 후 재시도 |
| 다른 사용자 데이터가 같이 보임 | Row Level Security(RLS, 사용자별로 데이터를 자동 분리하는 DB 보호 장치) 미설정 | "로그인한 사용자 본인 데이터만 보이도록 RLS를 설정해줘"라고 요청 |
