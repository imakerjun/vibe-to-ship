# 스텝 2: 시간 기반 학습 로드맵

<span className="step-tools">읽기 약 5분</span>

<Callout type="warning">
**욕심내지 마세요.** 한 번에 하나씩, 1주에 **2~3시간** 정도만 투자해도 충분합니다.
아래 로드맵은 **상한**이 아니라 **최소 경로**입니다. 본인 페이스에 맞게 늘려도 됩니다.
</Callout>

## 1주차 — 습관화

**목표**: Claude Code를 **일상 도구**로 만들기. 기능 확장 전에 "쓰는 습관"부터.

**추천 과제**
- 매일 **한 번은** Claude Code를 켭니다. 작아도 됩니다: "이 파일 무슨 내용이야?" 정도면 충분.
- 오늘 만든 저장소의 **CLAUDE.md** 를 한 번 더 읽고, 빠진 내용 1~2줄 추가합니다.
- 마주친 에러 메시지를 복사해서 Claude Code에 붙여넣고 **"무슨 의미야? 어떻게 고쳐?"** 물어봅니다.

**왜 이것부터**: 도구 확장 전에 **사용 리듬**을 만드는 것이 먼저입니다. 아무리 좋은 훅·스킬을 만들어도 Claude Code를 켜지 않으면 소용이 없습니다.

**공식 문서**: [Claude Code Quickstart](https://docs.claude.com/en/docs/claude-code/quickstart)

## 2주차 — 반복 줄이기

**목표**: "이 프롬프트 또 써야 하는데..." 싶은 것 **1개**를 슬래시 커맨드로 만들기.

**추천 과제**
1. 이번 주에 Claude Code에 **두 번 이상 비슷하게 요청한 것**을 메모합니다.
2. `.claude/commands/{이름}.md` 파일을 만들고 그 프롬프트를 적습니다.
3. 다음에는 `/이름` 한 줄로 호출합니다.

**예시 아이디어**
- `/주간보고` — "이번 주 커밋 내역을 요약해서 주간 보고용으로 정리해줘"
- `/한글화` — "다음 영어 텍스트를 자연스러운 한국어로 번역해줘"
- `/점검` — "이 파일의 오탈자와 어색한 문장을 찾아줘"

**공식 문서**: [Slash commands](https://docs.claude.com/en/docs/claude-code/slash-commands)

## 1달차 — 훅으로 자동화 맛보기

**목표**: **사람이 누르던 버튼**을 **자동으로 눌리는 버튼**으로 바꾸기. **1개만** 시도합니다.

**추천 과제**
- `PostToolUse` 훅으로 "파일 저장 후 자동 포맷팅" 또는 "자동 커밋 메모 작성" 중 하나만 시도해보세요.
- 처음엔 동작이 낯설 수 있으니, 실패해도 좋다는 마음으로 **작은 저장소**에서 실험하세요.

**왜 한 달 뒤인가**: 훅은 "자동 실행"이기 때문에 잘못 설정하면 의도치 않은 결과가 나올 수 있습니다. 1~2주차에 Claude Code에 익숙해진 후에 시도하는 것이 안전합니다.

**공식 문서**: [Hooks guide](https://docs.claude.com/en/docs/claude-code/hooks)

## 3달차 — 스킬 & MCP 도전

**목표**: **외부 세계**와 Claude Code를 연결하기.

**추천 과제 (둘 중 하나만 골라서)**
- **스킬(Skills)**: 내가 자주 하는 특정 도메인 작업(예: "보고서 작성", "회의록 정리")의 절차를 스킬로 정리합니다. 조건이 맞으면 Claude가 자동으로 참조합니다.
- **MCP 서버 1개**: Notion, Slack, Google Drive 중 **하나만** 골라 연결합니다. Claude가 그 서비스의 데이터를 직접 읽고 쓸 수 있게 됩니다.

**왜 3달 뒤인가**: 스킬과 MCP는 하네스 중에서도 **고급 기능**입니다. 기초(CLAUDE.md, 커맨드, 훅)가 체화된 뒤에 시도해야 효과를 제대로 느낄 수 있습니다.

**공식 문서**:
- [Skills](https://docs.claude.com/en/docs/claude-code/skills)
- [Model Context Protocol](https://docs.claude.com/en/docs/claude-code/mcp)

## 그 이후 — 팀 자산으로 공유

3달 뒤부터는 **혼자** 쓰는 하네스에서 **팀** 이 공유하는 하네스로 확장합니다.

- 내가 만든 **CLAUDE.md**, **커스텀 커맨드**, **스킬**을 팀 저장소에 올리고 동료에게 공유
- 팀 전용 **Playbook** 을 만들어 신규 입사자가 바로 쓸 수 있게
- 우아한형제들의 "AI 자산화" 사례처럼, 팀 전체가 AI를 **같은 방식**으로 쓰도록 환경 정리

## 이 로드맵을 벗어나도 괜찮은 이유

<Callout type="info">
**이 로드맵은 "정답"이 아니라 "참고 지도"입니다.**
- 1주차에 훅을 만들고 싶다면 만들어도 됩니다. 재미있으면 계속하는 게 최고의 학습입니다.
- 3개월이 지나도 CLAUDE.md만 잘 쓰고 있다면 그것도 충분히 좋은 상태입니다.
- **중요한 건 속도가 아니라 방향**입니다. "프롬프트 하나 잘 쓰는 것"에서 "환경을 설계하는 것"으로 관점이 이동했다면, 이미 성공한 것입니다.
</Callout>
