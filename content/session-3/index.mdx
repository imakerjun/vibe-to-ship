# 세션 3: 워크숍 이후의 길

<Callout type="info" emoji="🎯">
**이 세션의 목적** — 오늘 만든 앱은 "끝"이 아니라 **출발점** 입니다.
앞으로 여러분이 AI를 더 잘 쓰기 위해 할 일은 **프롬프트를 더 잘 쓰는 것**이 아니라,
**AI가 일할 작업실을 직접 설계하는 것**입니다. 이것을 **하네스 엔지니어링**이라 부릅니다.
</Callout>

## 이 세션의 사용법

세션 1·2가 **실습 중심**이었다면, 세션 3은 **읽기 자료 + 로드맵**입니다.
지금 바로 다 이해하려 하지 마세요. 오늘은 **북마크**만 해두고, 다음 주·다음 달·3달 후에 돌아와서 하나씩 시도하면 됩니다.

<Callout type="default">
**읽는 데 걸리는 시간**: 약 15분.
**오늘 해야 할 일**: 읽고 마음에 드는 섹션 1~2개를 북마크. **실습은 월요일부터.**
</Callout>

## 여러분은 이미 시작했습니다

혹시 "나는 비개발자인데 이런 걸 할 수 있을까?" 하는 생각이 드셨다면, 아래를 떠올려보세요.

- **세션 2 스텝 4**에서 `/init`으로 **CLAUDE.md** 를 만들었다면 — 여러분은 이미 **Context Engineering** 을 시작한 것입니다.
- **스코핑 템플릿**에 "누가 쓰나, 입력, 출력, 저장"을 적었다면 — 여러분은 이미 **Plan-based Execution** 을 경험한 것입니다.
- **오늘 만든 앱을 동료에게 URL로 공유**했다면 — 여러분은 이미 **팀 자산화** 의 첫 발을 디딘 것입니다.

**하네스 엔지니어링은 거창한 용어지만, 시작점은 여러분이 오늘 이미 해본 것들입니다.**
