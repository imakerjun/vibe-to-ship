# 스텝 2: DB 연동

<span className="step-tools">Lovable Cloud</span>

**이 스텝에서 하는 일**: 체크리스트 데이터를 DB에 저장해서 새로고침해도 유지되게 만듭니다.

<Callout type="info">
**Lovable Cloud**는 Lovable에 내장된 데이터베이스·인증·스토리지 플랫폼입니다.
별도 세팅 없이 대화만으로 PostgreSQL DB를 연결할 수 있습니다.
</Callout>

## 진행 단계

<Steps>

### DB 연동 요청하기

아래 프롬프트를 입력하세요.

```
Lovable Cloud를 사용해서 데이터를 저장해줘.

1. 점검 항목 테이블:
   - 항목명, 카테고리, 체크 여부, 메모, 수정일시

2. 현재 하드코딩된 샘플 데이터를 DB에 넣어줘

3. 체크박스를 누르면 DB에 저장되게 해줘

4. 새로고침해도 체크 상태가 유지되어야 해
```

### Cloud 활성화 허용하기

프롬프트를 보내면 아래와 같은 **Enable Cloud** 안내 화면이 나타납니다.
**Allow** 버튼을 클릭해서 Cloud 기능을 활성화하세요.

<img src="/images/lovable-cloud-enable.png" alt="Lovable Cloud Enable 안내 화면 — Allow 버튼" style={{maxWidth: '560px', width: '100%', borderRadius: '12px', border: '1px solid #333', marginTop: '1rem'}} />

<Callout type="warning">
**한 번 활성화하면 되돌릴 수 없습니다.** ("This can't be undone once enabled.")
워크숍에서는 **Asia Pacific** 지역을 그대로 두고 **Allow**를 누르면 됩니다.
</Callout>

활성화가 끝나면 Lovable이 알아서 DB 테이블을 생성하고 코드를 연결합니다.

### 생성된 데이터베이스 확인하기

DB가 잘 만들어졌는지 직접 확인해봅니다.

**1단계**: 상단 메뉴에서 **구름 모양 Cloud 버튼**을 클릭한 뒤, 좌측 사이드바에서 **Database**를 선택합니다.

<img src="/images/lovable-cloud-database-tab.png" alt="Lovable 상단 Cloud 버튼 → 좌측 Database 메뉴 클릭" style={{maxWidth: '720px', width: '100%', borderRadius: '12px', border: '1px solid #333', marginTop: '1rem'}} />

**2단계**: 생성된 테이블(예: `check_items`)을 클릭하면 실제로 저장된 데이터를 확인할 수 있습니다.

<img src="/images/lovable-cloud-database-rows.png" alt="check_items 테이블의 6개 행 데이터 — id, title, category, checked, memo 컬럼" style={{maxWidth: '720px', width: '100%', borderRadius: '12px', border: '1px solid #333', marginTop: '1rem'}} />

<Callout type="info">
**여기서 보이는 것들**
- 프롬프트로 요청한 샘플 데이터 6건이 실제로 저장되어 있습니다
- 각 컬럼(id / title / category / checked / memo)이 자동으로 생성되었습니다
- 이 화면에서 직접 데이터를 추가/수정/삭제할 수도 있습니다 (셀 더블클릭)
</Callout>

### 동작 확인하기

1. 앱 프리뷰로 돌아가서 항목을 체크합니다
2. 브라우저를 **새로고침**합니다
3. 체크 상태가 유지되는지 확인합니다
4. Database 화면에서 **Refresh**를 누르면 `checked` 값이 `true`로 바뀐 것을 확인할 수 있습니다

### 관리 기능 추가하기

```
관리자 기능을 추가해줘.

1. "항목 추가" 버튼: 새로운 점검 항목을 입력하는 폼
   - 항목명, 카테고리 선택, 저장 버튼

2. 기존 항목 삭제 기능: 각 카드에 삭제 버튼

3. 메모 수정: 메모란에 입력하면 자동 저장
```

</Steps>

---

## 확인 포인트

> 새로고침 후에도 체크 상태와 메모가 유지되고, 항목을 추가/삭제할 수 있으면 성공입니다.

## 막혔을 때

| 증상 | 해결 |
|------|------|
| DB 연동 후 데이터가 안 보임 | "DB에서 데이터를 불러오는 부분을 확인해줘"라고 요청 |
| 새로고침하면 데이터 사라짐 | "저장 로직이 제대로 동작하는지 확인해줘"라고 요청 |
| Lovable Cloud 활성화 안내 | Lovable 좌측 패널에서 Cloud 탭 확인 |

## 왜 이걸 하는가

> 스텝 1의 앱은 새로고침하면 모든 데이터가 사라집니다.
> DB를 연결하면 **데이터가 영구 저장**되어 실무에서 쓸 수 있는 앱이 됩니다.
