# 스텝 1: Lovable 프로토타이핑

<span className="step-tools">Lovable · 30분</span>

**이 스텝에서 하는 일**: Lovable에 프롬프트를 입력해서 웹앱의 첫 버전을 만듭니다.

<Steps>

### Lovable 접속하기

[lovable.dev](https://lovable.dev)에 접속합니다.
워크숍 워크스페이스에서 작업하고 있는지 확인하세요.

### 프롬프트 입력하기

아래 프롬프트를 복사해서 입력하세요.

```
업무 점검 체크리스트 웹앱을 만들어줘.

기능:
- 점검 항목을 카드 형태로 보여줘
- 각 항목에 체크박스와 메모 입력란
- 상단에 진행률 표시 (완료/전체)
- 점검 카테고리별 필터 (전체, 완료, 미완료)

샘플 데이터:
- 카테고리 "월간 점검": 고객정보 접근권한 확인, 비밀번호 변경 여부, 문서 보관 상태
- 카테고리 "분기 점검": 시스템 로그 점검, 외부감사 자료 준비, 규정 변경사항 반영

디자인:
- 깔끔한 카드 UI, 다크 모드
- 모바일 반응형
- 상단에 "OK금융 업무 점검" 타이틀
```

### 결과 확인하기

Lovable이 앱을 생성하면 프리뷰에서 확인합니다. 아래와 비슷한 화면이 나오면 성공입니다.

<img src="/images/lovable-result-checklist.png" alt="Lovable로 생성된 OK금융 업무 점검 체크리스트 결과 화면" style={{maxWidth: '720px', width: '100%', borderRadius: '12px', border: '1px solid #333', marginTop: '1rem'}} />

- 체크리스트 카드가 보이나요?
- 체크박스를 클릭하면 진행률이 바뀌나요?
- 카테고리 필터(전체/완료/미완료)가 보이나요?

<Callout type="info">
**완전히 똑같지 않아도 괜찮습니다.** Lovable이 매번 조금씩 다른 디자인을 생성합니다.
중요한 건 *체크리스트가 동작하는지*입니다.
</Callout>

### 수정 요청하기

최소 1가지는 본인이 직접 수정 요청해보세요.

<Callout type="default" emoji="✏️">
**수정 요청 예시:**
- "완료된 항목은 색상을 연하게 해줘"
- "카테고리별 진행률도 따로 보여줘"
- "점검 항목을 접었다 펼 수 있게 해줘"
</Callout>

</Steps>

---

## 확인 포인트

> 체크리스트 앱이 프리뷰에서 동작하고, 수정 요청이 반영되었으면 성공입니다.

## 막혔을 때

| 증상 | 해결 |
|------|------|
| Lovable이 에러를 표시함 | 채팅에 "에러를 수정해줘"라고 입력 |
| 프리뷰가 빈 화면 | 브라우저 새로고침 후 다시 확인 |
| 디자인이 마음에 안 듦 | 구체적으로 요청 (예: "카드 사이 간격을 넓혀줘") |

## 왜 이걸 하는가

> Lovable은 대화만으로 웹앱의 첫 버전을 만들어줍니다.
> 이 스텝의 목적은 **빠르게 동작하는 것을 만드는 것**입니다.
> 완성도는 이후 스텝에서 올립니다.
