# 스텝 4: GitHub 연결 + 로컬 환경 준비

<span className="step-tools">GitHub · VS Code · Claude Code</span>

**이 스텝에서 하는 일**: Lovable 코드를 GitHub에 연결하고, 내 컴퓨터로 가져와 로컬에서 실행할 수 있는 상태까지 만듭니다. (개선과 배포는 다음 스텝에서 진행합니다.)

## GitHub가 뭔가요?

코드를 직접 다루기 전에, **Git**과 **GitHub**가 무엇인지 간단히 알아봅니다.

<Callout type="default">
**Git** — 파일의 변경 이력을 기록하는 도구입니다. "저장" 버튼을 누르듯이, 코드의 각 시점을 스냅샷으로 남깁니다. 실수하면 이전 상태로 돌아갈 수 있습니다.

**GitHub** — Git으로 관리하는 코드를 인터넷에 보관하는 서비스입니다. Google Drive가 문서를 클라우드에 저장하듯, GitHub는 코드를 클라우드에 저장합니다. 여러 사람이 같은 코드를 함께 수정할 수도 있습니다.
</Callout>

이번 워크숍에서 GitHub는 **Lovable과 내 컴퓨터 사이의 다리** 역할을 합니다.

<Mermaid chart={`flowchart LR
    L["Lovable<br/>(웹에서 만들기)"] -->|코드 저장| G["GitHub<br/>(클라우드 보관소)"]
    G -->|코드 가져오기| PC["내 컴퓨터<br/>(Claude Code로 개선)"]
    PC -->|수정 반영| G
    G -->|자동 반영| L

    style L fill:#fef3c7,stroke:#f59e0b
    style G fill:#e0e7ff,stroke:#6366f1
    style PC fill:#dbeafe,stroke:#3b82f6
`} />

<Callout type="warning">
**왜 코드를 가져오나요?**

Lovable은 빠른 프로토타이핑에 최적화되어 있지만, Lovable 안에서만 수정할 수 있습니다.
코드를 내 컴퓨터로 가져오면 Claude Code로 원하는 만큼 정밀하게 다듬을 수 있습니다.

자세한 비교는 [왜 Claude Code인가](/reference/why-claude-code)를 참고하세요.
</Callout>

<Steps>

### Lovable에서 GitHub 연결하기

Lovable과 GitHub를 연결하면 코드가 양방향으로 동기화됩니다.

<Mermaid chart={`sequenceDiagram
    participant L as Lovable
    participant G as GitHub
    participant PC as 내 컴퓨터

    L->>G: Connect GitHub (저장소 생성)
    Note over L,G: 양방향 동기화 시작
    L-->>G: Lovable에서 수정 → 자동 반영
    G-->>L: GitHub에서 수정 → Lovable 반영
    G->>PC: 코드 가져오기
    PC->>G: 수정한 코드 올리기
    G-->>L: Lovable에도 반영
`} />

<Callout type="warning">
**GitHub 계정이 없는 경우**

워크숍 전 미리 준비하도록 안내했지만, 아직 계정이 없다면 지금 만드세요.

1. [github.com](https://github.com)에 접속합니다
2. **Sign up**을 클릭하고 이메일·비밀번호·사용자명을 입력합니다
3. 이메일 인증을 완료하면 바로 사용할 수 있습니다

계정 생성은 **5분** 이내에 완료됩니다. 인증 이메일이 오지 않으면 스팸 폴더를 확인하세요.
</Callout>

1. Lovable 상단 메뉴에서 **GitHub 아이콘**을 클릭합니다

<img src="/images/lovable-github-connect.png" alt="Lovable GitHub 연결 버튼" style={{maxWidth: '480px', borderRadius: '12px', border: '1px solid #333'}} />

2. **Connect to GitHub**를 선택합니다
3. 저장소 이름을 입력하고 **Create Repository**를 클릭합니다

<Callout type="info">
Lovable의 GitHub 연결은 **양방향 동기화**됩니다.
Lovable에서 수정하면 GitHub에 반영되고, GitHub에서 수정하면 Lovable에도 반영됩니다.
</Callout>

### 저장소 주소 복사하기

GitHub에 연결되면 아래와 같이 저장소 주소를 확인할 수 있는 화면이 나옵니다.
**Clone** 영역의 **HTTPS** 탭에서 주소를 복사해두세요. 다음 단계에서 Claude Code에 붙여넣을 때 사용합니다.

<img src="/images/lovable-github-clone-url.png" alt="Lovable GitHub 연결 화면 — Clone HTTPS URL 복사 위치" style={{maxWidth: '800px', width: '100%', borderRadius: '12px', border: '1px solid #e5e7eb', marginTop: '1rem'}} />

<Callout type="info">
**어디서 이 화면을 볼 수 있나요?**
Lovable 상단 메뉴의 **GitHub 아이콘**을 다시 클릭하면 연결된 저장소 정보 화면이 나타납니다.
Clone 섹션의 오른쪽 **복사 아이콘**을 클릭하면 URL이 클립보드에 복사됩니다.
</Callout>

### VS Code + Claude Code 설치하기

코드를 내 컴퓨터에서 다루려면 **에디터**가 필요합니다.
VS Code는 무료 코드 편집기이고, 여기에 Claude Code 확장을 설치하면 대화로 코드를 수정할 수 있습니다.

<Callout type="info">
**이미 VS Code와 Claude Code가 설치되어 있다면?**

1단계와 2단계를 건너뛰고 바로 **3단계: Claude Code 첫 실행과 로그인**으로 이동하세요.
설치 여부는 다음으로 확인합니다.

- VS Code: 앱 목록에서 "Visual Studio Code"가 있으면 설치됨
- Claude Code: VS Code 실행 후 왼쪽 사이드바에 Claude 아이콘(C)이 있으면 설치됨
</Callout>

**1단계: VS Code 설치**

[code.visualstudio.com](https://code.visualstudio.com)에 접속해서 다운로드 → 설치합니다.

**2단계: Claude Code 확장 설치**

1. VS Code를 열고 왼쪽 사이드바에서 **확장(Extensions)** 아이콘(네모 4개)을 클릭합니다
2. 검색창에 `claude code`를 입력합니다
3. 게시자가 **Anthropic**인 "Claude Code for VS Code"를 찾아 **Install**을 클릭합니다
4. 설치 후 왼쪽 사이드바에 Claude 아이콘이 나타납니다

<img src="/images/vscode-claude-code-extension.png" alt="VS Code 확장 마켓플레이스에서 Claude Code for VS Code 검색 — 게시자 Anthropic" style={{maxWidth: '800px', width: '100%', borderRadius: '12px', border: '1px solid #333', marginTop: '1rem'}} />

<Callout type="warning">
**반드시 게시자가 `Anthropic`인 것을 설치하세요.** "Claude Code" 이름을 쓰는 비공식 확장이 여러 개 있습니다.
첫 번째 결과의 이름이 <strong>"Claude Code for VS Code"</strong>이고 게시자가 <strong>Anthropic</strong>인지 확인하세요.
</Callout>

**3단계: Claude Code 첫 실행과 로그인**

확장 설치가 끝났으면 Claude Code에 로그인합니다. 공식 문서의 [Quickstart](https://docs.claude.com/en/docs/claude-code/quickstart) 흐름을 따릅니다.

1. VS Code 왼쪽 사이드바에서 **Claude 아이콘**(C)을 클릭해 Claude Code 패널을 엽니다
2. 처음 열면 로그인 안내가 나타납니다. **Sign in**을 클릭하면 브라우저가 열리고 Anthropic 계정으로 로그인합니다
3. 로그인이 끝나면 VS Code로 돌아와 Claude Code 채팅창이 활성화됩니다

<Callout type="info">
**워크숍 계정으로 로그인하세요.** 개인 Anthropic 계정이 없다면 워크숍에서 제공하는 계정을 사용합니다. 브라우저 팝업이 차단되면 주소창 오른쪽 아이콘에서 허용해 주세요.
</Callout>

### GitHub에서 코드 가져오기

Claude Code에게 직접 요청합니다. VS Code 왼쪽의 **Claude 아이콘**을 클릭하고 채팅창에 입력하세요.

```
GitHub에서 내 저장소를 클론해줘.
저장소 주소는 https://github.com/{username}/{repo-name} 이야.
```

<Callout type="default">
`{username}`과 `{repo-name}`은 GitHub 저장소 페이지에서 확인할 수 있습니다.
이전 단계에서 복사해둔 Clone HTTPS URL을 그대로 붙여넣어도 됩니다.
</Callout>

Claude Code가 알아서 코드를 가져오고, 필요한 설정을 진행합니다.

<Callout type="default">
**체크포인트: 폴더 구조 확인**

코드를 가져온 후 VS Code 왼쪽 탐색기(파일 아이콘)에서 다음 항목이 보이면 정상입니다.

- `src/` — 앱의 핵심 소스 코드가 들어 있는 폴더
- `package.json` — 프로젝트 정보와 의존성 목록
- `index.html` — 앱의 시작점 HTML 파일

이 항목들이 보이지 않으면 클론이 완료되지 않은 것입니다. Claude Code 채팅창에 "클론이 잘 됐는지 확인해줘"라고 입력하세요.
</Callout>

### 프로젝트 컨텍스트 초기화 (`/init`)

클론이 끝났으면 Claude Code 채팅창에 슬래시 명령을 입력합니다.

```
/init
```

`/init`은 Claude Code가 현재 프로젝트를 훑어보고 [`CLAUDE.md`](https://docs.claude.com/en/docs/claude-code/memory) 라는 메모리 파일을 생성하는 명령입니다. 이 파일에는 프로젝트 구조, 사용 언어, 빌드 명령 같은 정보가 요약되어 저장되며, 이후 모든 대화에 자동으로 참고되어 답변 품질이 올라갑니다.

<Callout type="default">
**체크포인트: `CLAUDE.md` 생성 확인**

VS Code 왼쪽 탐색기에서 프로젝트 루트에 `CLAUDE.md` 파일이 새로 생겼는지 확인하세요. 내용은 열어보고 어떻게 요약되었는지 한 번 훑어보면 좋습니다.
</Callout>

### 로컬에서 앱 실행하기

이제 내 컴퓨터에서 앱을 직접 실행해봅니다. Claude Code에게 요청하세요.

```
이 프로젝트를 로컬에서 실행해줘.
```

Claude Code가 필요한 패키지를 설치(`npm install`)하고, 개발 서버를 실행(`npm run dev`)한 뒤, 접속 주소를 알려줍니다. 보통 아래와 같은 주소가 나옵니다.

```
http://localhost:5173
```

<Callout type="info">
**localhost가 뭔가요?**
`localhost`는 **내 컴퓨터 자체**를 가리키는 주소입니다. 즉, 지금 실행한 앱은 인터넷이 아니라 **내 컴퓨터 안에서만** 동작합니다. 나만 접속할 수 있어서 테스트하기 안전합니다.
뒤의 숫자(예: `5173`)는 **포트 번호**로, 여러 프로그램이 같은 컴퓨터에서 겹치지 않도록 구분하는 번호입니다.
</Callout>

브라우저 새 탭을 열고 그 주소로 접속하면 Lovable에서 만든 앱이 동일하게 동작합니다.

<Callout type="default">
**체크포인트: 앱 동작 확인**

- 브라우저에서 `http://localhost:5173`에 접속했을 때 앱 화면이 보이나요?
- Lovable 프리뷰에서 보이던 것과 같은 모습인가요?
- 구글 로그인과 체크리스트가 그대로 동작하나요?

**세 가지 모두 Yes면 로컬 개발 환경이 정상적으로 준비된 것입니다.**
</Callout>

<Callout type="warning">
**개발 서버는 계속 실행 상태여야 합니다.**
Claude Code가 실행한 `npm run dev`는 **서버**입니다. VS Code 하단 터미널에 실행 로그가 계속 남아 있어야 브라우저에서 접속할 수 있습니다.
터미널을 닫거나 Claude Code에게 "종료해줘"라고 요청하면 `localhost`에서 더 이상 접속되지 않습니다.
</Callout>

</Steps>

## 막혔을 때

| 증상 | 해결 |
|------|------|
| VS Code 설치가 안 됨 | [code.visualstudio.com](https://code.visualstudio.com)에서 직접 다운로드 |
| Claude Code 확장이 안 보임 | 확장 검색에서 "Claude Code" 입력, 게시자가 "Anthropic"인지 확인 |
| Claude Code 로그인 실패 | 브라우저 팝업 차단 확인, 워크숍 제공 계정으로 로그인 |
| 코드 가져오기 실패 | GitHub 로그인 확인, 저장소 URL 확인 |
| Claude Code 실행 안 됨 | [설치 가이드](/reference/claude-code-install) 참고 |
