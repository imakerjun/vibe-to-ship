# 스텝 3: 구글 로그인

<span className="step-tools">Lovable Cloud · 20분</span>

**이 스텝에서 하는 일**: Lovable Cloud의 Managed Google Auth로 구글 로그인을 추가하고, 사용자별로 데이터를 분리합니다.

<Callout type="info">
**Lovable Cloud의 Managed Google Auth**

Lovable Cloud는 구글 OAuth를 자체 관리합니다.
Google Cloud 콘솔에서 별도로 OAuth 클라이언트를 만들 필요 없이,
프롬프트 한 줄이면 구글 로그인이 활성화됩니다.
</Callout>

<Steps>

### 구글 로그인 추가하기

아래 프롬프트를 입력하세요.

```
구글 로그인 기능을 추가해줘.

1. 로그인하지 않은 사용자에게는 로그인 버튼만 보여줘
2. 로그인한 사용자만 체크리스트를 사용할 수 있게 해줘
3. 사용자별로 체크 데이터가 분리되어야 해
4. 상단에 로그인한 사용자 이름과 프로필 사진 표시
5. 로그아웃 버튼도 추가해줘
```

### 로그인 테스트하기

1. 프리뷰에서 **구글 로그인** 버튼을 클릭합니다
2. 본인의 구글 계정으로 로그인합니다
3. 로그인 후 체크리스트가 정상적으로 보이는지 확인합니다

### 사용자별 데이터 분리 확인하기

1. 몇 개 항목을 체크합니다
2. **로그아웃**합니다
3. 로그인 화면만 보이는지 확인합니다
4. 다시 **로그인**합니다
5. 아까 체크한 항목이 그대로 유지되는지 확인합니다

<Callout type="default" emoji="✏️">
**추가 요청 예시:**
- "로그인 페이지에 '업무 점검 시작하기' 안내 문구를 추가해줘"
- "로그아웃할 때 확인 팝업을 보여줘"
</Callout>

</Steps>

---

## 확인 포인트

> 구글 로그인 후 체크리스트가 보이고, 로그아웃하면 로그인 화면으로 돌아가면 성공입니다.

## 막혔을 때

| 증상 | 해결 |
|------|------|
| 구글 로그인 버튼이 안 보임 | "구글 로그인 버튼이 안 보이는데 확인해줘"라고 요청 |
| 로그인 후 빈 화면 | "로그인 후 데이터를 불러오는 부분을 확인해줘"라고 요청 |
| 로그인 팝업이 닫히지 않음 | 브라우저 팝업 차단 해제 후 다시 시도 |
| "auth 설정 오류" 메시지 | "인증 설정을 확인해줘"라고 요청 |

## 왜 이걸 하는가

> 스텝 2까지는 누구나 같은 데이터를 보고 수정할 수 있었습니다.
> 로그인을 추가하면 **누가 어떤 데이터를 수정했는지** 알 수 있고,
> 사용자마다 자신의 체크리스트를 관리할 수 있습니다.
> 실무에서 쓸 수 있는 앱이 되려면 인증은 필수입니다.
