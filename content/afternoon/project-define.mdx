# 프로젝트 정의 (30분)

## 스코핑 템플릿

조별로 아래 템플릿을 채워주세요. **오늘 안에 가능한 범위**에 집중합니다.

```
[앱 이름]:

사용자: 누가 쓰는가?
입력: 무엇을 넣는가?
출력: 무엇이 나오는가?
DB에 저장할 것: 무엇인가?
오늘 우선 만들 것: ___만 먼저 만든다
```

## 예시

```
[앱 이름]: 부서 점검 현황판

사용자: 감사팀 팀원 + 팀장
입력: 점검 항목 체크, 메모 입력
출력: 부서별 점검 완료율 대시보드
DB에 저장할 것: 점검 결과, 메모, 담당자
오늘 우선 만들 것: 체크리스트 + 완료율 표시만 먼저 만든다
```

<Callout type="warning">
**범위를 좁히세요!**

"전부 다 만들기"보다 "핵심 1가지를 완성"하는 게 훨씬 낫습니다.
오늘 배포까지 가는 것이 목표입니다.
</Callout>

## 프로젝트 아이디어 (사전 설문 기반)

참여자분들이 설문에서 적어주신 내용을 모았습니다.

| 유형 | 예시 |
|------|------|
| **점검/감사** | 자점감사 체크리스트, 법인카드 점검, 규정 변경 확인 |
| **데이터 취합** | 인력 현황 집계, 부서별 업무 현황, 성과 평가 취합 |
| **보고서 자동화** | 해외법인 보고서 번역, 감사 결과 보고서, 실적 대시보드 |
| **업무 도구** | 여신금리 계산기, 회의실 예약, FAQ 검색 |

조원들의 아이디어 중 **가장 간단하게 MVP를 만들 수 있는 것** 1개를 고르세요.
