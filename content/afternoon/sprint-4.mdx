# 스프린트 4: 한 단계 더 (50분)

스프린트 3까지 만든 앱에서 **가장 아쉬운 1가지**를 Claude Code 실전 기법으로 해결합니다.

## CLAUDE.md 작성하기

Claude Code에게 "내 프로젝트의 맥락"을 파일로 알려주면, 매번 설명하지 않아도 됩니다.

```
이 프로젝트의 CLAUDE.md 파일을 만들어줘.
프로젝트의 목적, 기술 스택, 주요 파일 구조를 정리해줘.
```

<Callout type="info">
**CLAUDE.md**는 Claude Code에게 "나는 이렇게 일한다"를 알려주는 파일입니다.
한 번 작성하면 매번 반복해서 설명하지 않아도 됩니다.
</Callout>

## 실전 기법 적용하기

아래에서 본인에게 필요한 것을 골라 적용해보세요.

### 기능 확장

```
{구체적으로 원하는 기능}을 추가해줘.
기존 코드 구조를 유지하면서 구현해줘.
```

### UI/UX 개선

```
사용자가 처음 접속했을 때 어떻게 쓰는지 알 수 있도록
온보딩 가이드나 빈 상태 안내를 추가해줘.
```

### 코드 품질 개선

```
이 프로젝트에서 중복된 코드나 개선할 수 있는 부분을 찾아서 리팩터링해줘.
```

### 데이터 내보내기

```
현재 데이터를 CSV나 PDF로 다운로드할 수 있는 기능을 추가해줘.
```

## 최종 배포

개선이 끝나면 최종 배포합니다.

```
변경 사항을 커밋하고 배포해줘.
```
