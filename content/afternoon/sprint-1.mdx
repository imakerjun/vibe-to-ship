# 스프린트 1: Lovable MVP (45분)

오전 Step 1과 동일한 흐름입니다. 이번에는 **프롬프트를 직접 작성**합니다.

## 프롬프트 작성 가이드

오전에 사용한 프롬프트 구조를 참고하세요.

```
{앱 이름} 웹앱을 만들어줘.

기능:
- {핵심 기능 1}
- {핵심 기능 2}
- {핵심 기능 3}

샘플 데이터:
- {실제와 비슷한 예시 데이터}

디자인:
- 깔끔한 카드 UI, 다크 모드
- 모바일 반응형
```

<Callout type="info">
**팁**: 기능은 3개 이하로 시작하세요.
Lovable에게 한 번에 많이 요청하면 복잡해집니다.
핵심 1개를 먼저 만들고, 채팅으로 하나씩 추가하세요.
</Callout>

## 진행 순서

1. 스코핑 템플릿을 바탕으로 프롬프트 작성 (10분)
2. Lovable에 입력하여 첫 버전 생성 (10분)
3. 수정 요청 2~3회 (15분)
4. 조원에게 프리뷰 보여주고 피드백 (10분)

## 확인 포인트

> 앱의 핵심 기능 1개가 프리뷰에서 동작하면 스프린트 1 완료입니다.
