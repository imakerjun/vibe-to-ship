# 스프린트 1: 문제 탐색 + Lovable MVP (50분)

## 문제 탐색 (15분)

조별로 아래 스코핑 템플릿을 채워주세요. **오늘 안에 가능한 범위**에 집중합니다.

```
[앱 이름]:

사용자: 누가 쓰는가?
입력: 무엇을 넣는가?
출력: 무엇이 나오는가?
DB에 저장할 것: 무엇인가?
오늘 우선 만들 것: ___만 먼저 만든다
```

### 프로젝트 아이디어 (사전 설문 기반)

| 유형 | 예시 |
|------|------|
| **점검/감사** | 자점감사 체크리스트, 법인카드 점검, 규정 변경 확인 |
| **데이터 취합** | 인력 현황 집계, 부서별 업무 현황, 성과 평가 취합 |
| **보고서 자동화** | 해외법인 보고서 번역, 감사 결과 보고서, 실적 대시보드 |
| **업무 도구** | 여신금리 계산기, 회의실 예약, FAQ 검색 |

<Callout type="warning">
**범위를 좁히세요!** "전부 다 만들기"보다 "핵심 1가지를 완성"하는 게 훨씬 낫습니다.
</Callout>

## Lovable MVP (35분)

오전 Step 1과 동일한 흐름입니다. 이번에는 **프롬프트를 직접 작성**합니다.

### 프롬프트 작성 가이드

```
{앱 이름} 웹앱을 만들어줘.

기능:
- {핵심 기능 1}
- {핵심 기능 2}
- {핵심 기능 3}

샘플 데이터:
- {실제와 비슷한 예시 데이터}

디자인:
- 깔끔한 카드 UI, 다크 모드
- 모바일 반응형
```

<Callout type="info">
**팁**: 기능은 3개 이하로 시작하세요.
핵심 1개를 먼저 만들고, 채팅으로 하나씩 추가하세요.
</Callout>

## 확인 포인트

> 앱의 핵심 기능 1개가 프리뷰에서 동작하면 스프린트 1 완료입니다.
