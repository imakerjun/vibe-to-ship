# AX, 반복에서 축적으로

> 우아한형제들 사례 중심 | 삼성SDS 특강 (2026.03.25)

오늘 워크숍에서 경험한 "아이디어 → 동작하는 앱" 사이클은 시작일 뿐입니다.
AI 활용의 진짜 전환점은 **반복을 축적으로 바꾸는 구조**를 만드는 것입니다.

## 바이브 코딩 vs 에이전틱 엔지니어링

| | 바이브 코딩 | 에이전틱 엔지니어링 |
|---|---|---|
| **맥락 전달** | 그때그때 프롬프트 | 구조화된 문서 (CLAUDE.md 등) |
| **품질 기준** | "돌아가면 OK" | 팀의 기준과 동일 |
| **경험 축적** | 개인 대화에 남음 | 팀 자산으로 공유·재사용 |
| **적합한 상황** | 탐색, 프로토타입 | 팀 개발, 장기 유지보수 |

<Callout type="info">
오늘 세션 1에서 한 것이 **바이브 코딩**이고, 세션 2 스텝 4에서 CLAUDE.md와 커스텀 커맨드를 만든 것이 **에이전틱 엔지니어링의 첫걸음**입니다.
</Callout>

    B --> C["Level 3\\n조직 → 문화\\n누구나 쓸 수 있게"]

|------|--------|-----------|-----------|
| **Level 1** 개인 → 도구 | 반복을 자동화 | "내가 반복하는 AI 작업이 뭔가?" | Lovable + Claude Code 사이클 |
| **Level 2** 팀 → 자산 | 에센스를 구조화 | "이 노하우를 다른 사람이 재현할 수 있나?" | CLAUDE.md, 커스텀 커맨드 |
| **Level 3** 조직 → 문화 | 임계점을 넘기 | "축적된 자산 위에서 누구나 쓸 수 있나?" | 다음 단계 목표 |
