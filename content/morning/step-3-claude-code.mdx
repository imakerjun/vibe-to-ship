# Step 3: GitHub → Claude Code

<span className="step-tools">GitHub · Claude Code · 40분</span>

**이 단계에서 하는 일**: Lovable에서 만든 코드를 GitHub에 연결하고, 로컬에서 Claude Code로 개선합니다.

<Callout type="warning">
**왜 코드를 가져오나요?**

Lovable은 빠른 프로토타이핑에 좋지만, 세밀한 커스터마이징에는 한계가 있습니다.
코드를 로컬로 가져오면 Claude Code로 원하는 만큼 정밀하게 다듬을 수 있습니다.
</Callout>

<Steps>

### Lovable에서 GitHub 연결하기

1. Lovable 상단 메뉴에서 **GitHub** 아이콘을 클릭합니다
2. **Connect to GitHub**를 선택합니다
3. 저장소 이름을 입력하고 **Create Repository**를 클릭합니다

### 로컬로 가져오기

터미널을 열고 아래 명령어를 실행합니다. `{username}`과 `{repo-name}`은 본인 것으로 바꿔주세요.

```bash
git clone https://github.com/{username}/{repo-name}.git
cd {repo-name}
npm install
```

### Claude Code 실행하기

같은 폴더에서 Claude Code를 실행합니다.

```bash
claude
```

### 프로젝트 구조 파악하기

Claude Code에 아래와 같이 입력합니다.

```
이 프로젝트의 구조를 설명해줘.
주요 파일과 역할을 간단히 알려줘.
```

<Callout type="info">
**관찰 포인트**: Lovable이 만든 코드가 실제 파일로 존재합니다.
`src/` 폴더 안에 컴포넌트, 페이지, 유틸리티가 정리되어 있습니다.
</Callout>

### 기능 1건 추가하기

Claude Code에 개선 사항을 요청합니다.

```
점검 완료율을 날짜별로 보여주는 간단한 차트를 추가해줘.
최근 7일간의 완료율을 막대 그래프로 보여주면 좋겠어.
대시보드 상단에 배치해줘.
```

### 로컬에서 확인하기

```bash
npm run dev
```

브라우저에서 `http://localhost:5173`(또는 안내된 포트)에 접속하여 변경 사항을 확인합니다.

</Steps>

---

## 확인 포인트

> Claude Code가 추가한 차트(또는 기능)가 로컬 브라우저에서 보이면 성공입니다.

## 막혔을 때

| 증상 | 해결 |
|------|------|
| `git clone` 실패 | GitHub 로그인 확인, 저장소 URL 확인 |
| `npm install` 에러 | `node -v`로 Node.js 설치 확인 (18 이상 필요) |
| Claude Code 실행 안 됨 | `npm install -g @anthropic-ai/claude-code` 로 설치 |
| `npm run dev` 에러 | Claude Code에 "dev 서버 실행 시 에러가 나는데 수정해줘"라고 요청 |

## 왜 이걸 하는가

> Lovable은 **빠르게 만들기**, Claude Code는 **정밀하게 다듬기**입니다.
> 실무에서는 이 두 도구를 오가며 작업하게 됩니다.
> 코드를 가져오는 순간, 앱은 완전히 **내 것**이 됩니다.
