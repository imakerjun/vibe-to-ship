# Step 4: 배포 + 교차 체험

<span className="step-tools">배포 · 20분</span>

**이 단계에서 하는 일**: 완성된 앱을 배포하고, 조원에게 URL을 공유하여 직접 써보게 합니다.

<Steps>

### 배포하기

두 가지 방법 중 하나를 선택합니다.

**방법 A: Lovable에서 배포 (간단)**

Lovable 우상단 **Publish** 버튼 클릭 → URL 생성

**방법 B: Claude Code에서 배포 (코드 변경 포함)**

```
이 프로젝트를 배포해줘.
Vercel이나 Netlify 중 편한 방법으로.
```

### URL 공유하기

생성된 URL을 조원에게 공유합니다.

### 다른 사람 앱 써보기

조원의 URL에 접속해서 직접 써봅니다.

- 체크리스트 항목을 체크해보세요
- 새 항목을 추가해보세요
- 모바일에서도 열어보세요

</Steps>

---

## 확인 포인트

> 배포된 URL에서 앱이 동작하고, 조원이 실제로 사용할 수 있으면 성공입니다.

<Callout type="info" emoji="🎉">
**축하합니다!**

대화에서 시작한 아이디어가 → DB가 연결된 앱이 되었고 → 코드로 개선했고 → 누구나 접속할 수 있는 URL이 되었습니다.

이것이 **Lovable → DB → Claude Code → 배포** 풀사이클입니다.
오후에는 이 사이클을 **내 업무 과제**에 적용합니다.
</Callout>

## 왜 이걸 하는가

> 배포는 "내 컴퓨터에서만 돌아가는 것"을 "누구나 쓸 수 있는 것"으로 바꿉니다.
> 옆 사람이 내 앱을 실제로 쓰는 순간, 프로토타입이 **제품**이 됩니다.
